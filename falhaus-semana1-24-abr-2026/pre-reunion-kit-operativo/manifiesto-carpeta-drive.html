Kit operativo · Estructura Drive

Manifiesto de la carpeta de contenido

La estructura exacta que tiene que replicarse en Google Drive para que el sistema pre-reunión funcione desde el lunes. Cada carpeta tiene un propósito. Cada archivo un nombre estandarizado.

Estructura de carpetas

Crear en Drive una carpeta raíz llamada FALHAUS — Contenido Pre-Reunión. Dentro, replicar este árbol:

FALHAUS — Contenido Pre-Reunión/ │ ├─ 01_Videos/ # activos audiovisuales │ ├─ proceso/ # proceso constructivo (1-2 min) │ │ ├─ falhaus-proceso-entreRios-01min.mp4 │ │ └─ falhaus-proceso-generico-02min.mp4 │ │ │ └─ casos/ # video corto por caso │ ├─ caso-familiar-pilar-01min.mp4 │ └─ caso-inversor-pinamar-01min.mp4 │ ├─ 02_Casos/ # una carpeta por cada caso cerrado │ │ │ ├─ caso-01-familiar-pilar/ │ │ ├─ foto-antes-01.jpg │ │ ├─ foto-antes-02.jpg │ │ ├─ foto-despues-01.jpg │ │ ├─ foto-despues-02.jpg │ │ ├─ foto-despues-03.jpg │ │ ├─ ficha-caso.pdf # m² + tiempo + presupuesto │ │ └─ video-obra-01min.mp4 │ │ │ └─ caso-02-inversor-pinamar/ # misma estructura │ ├─ 03_Institucional/ # material formal para leads exigentes │ ├─ brochure-institucional-falhaus-2026.pdf │ ├─ catalogo-modelos-2026.pdf │ └─ ficha-tecnica-constructiva-2026.pdf │ ├─ 04_Plantillas-Mensajes/ # los 4 mensajes como .txt │ ├─ M1-T48-confirmacion.txt │ ├─ M2-T24-caso-exito.txt │ ├─ M3-T2h-recordatorio.txt │ └─ M4-reagendo.txt │ └─ 99_Links-Cortos/ # tabla con links bit.ly activos └─ links-activos.gsheet # nombre → URL corta → URL larga

Convención de nombres: todos los archivos kebab-case, sin tildes ni eñes (para que los links no rompan). Formato fecha en nombres donde aplique: YYYY-MM-DD.

Permisos de compartir

Cada archivo/carpeta que se comparte con un lead tiene que estar en modo "Cualquier persona con el enlace: Lector". Nunca "solo invitados" — el lead abandona si tiene que logear.

Archivos sensibles (ficha-caso.pdf con datos económicos, catálogo con precios) → carpeta de acceso controlado, se comparte solo cuando el lead pide explícitamente.
